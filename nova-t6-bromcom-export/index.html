Load the student list (optional)

To build class lists, load a CSV of students with the class codes they should be in — for example an export from SIMS (e.g. a Group Analysis / student report with teaching groups) or from your current MIS. Each student needs an identifier (UPN and/or admission number) and one or more class codes. A student’s classes can be in several columns, or in one column separated by commas/semicolons — both work. If you only want the timetable structure itself, you can skip this step.

Export files for Bromcom

Files download straight from your browser to this computer — nothing is uploaded anywhere. Import them into Bromcom via Data Import (or hand them to your Bromcom onboarding team). Column headers are standard, but do check them against your Bromcom import template and adjust the options below if your template differs.

1. Classes (teaching groups)

One row per class: name, subject, year group, block/band, main teacher, room and periods per cycle. Use this to create the classes in Bromcom.

2. Timetable (scheduled lessons)

One row per lesson: class, day, period, teacher, room. Only available if your Nova-T6 export included day/period columns.

3. Class lists (student memberships)

One row per student-per-class: UPN, admission number, name, year and the class they belong to. This is the file that allocates students to the new classes.

4. Exceptions report

Everything to chase before import day: unmatched student codes, students with no classes, classes with no students, and timetable clashes.
